Để deploy OpenClaw thì thường sẽ sử dụng một dịch vụ VPS (Virtual Private Server) để chạy nó, trực tiếp hoặc sử dụng Docker để chạy.

<h1 align="center">Setup OpenClaw trong môi trường Docker</h1>

Trong ví dụ này, em sẽ deploy OpenClaw trong Docker trên một VM chạy Ubuntu (chung VM với những ví dụ trước)

**Setup Docker trong VM**

Chạy script:
```
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
```

Test đã cài đặt được chưa:
```
sudo docker run hello-world
```

Note:

Xem danh sách images/containers: docker image/container ls

Xóa mọi container: docker rm $(docker ps –a –q)

docker container prune: xóa tất cả container đang không hoạt động

docker stop <tên container/container ID>: dừng container

docker stop $(docker ps –a –q): dừng mọi container

docker rmi <image ID>: xóa image

**Tạo image OpenClaw cho Docker**

Cần thực hiện git clone về repo source code của OpenClaw và chạy script setup cho docker.

```
git clone https://github.com/openclaw/openclaw.git 
cd openclaw
mkdir -p ~/.openclaw/workspace
chown -R 1000:1000 ~/.openclaw
./docker-setup.sh
```

Kết thúc 4/2: hiện vẫn đang fix bug khi khởi tạo image và container, hiện vẫn chưa fix được.

Lỗi 1: khi chạy ./docker-setup.sh để khởi tạo image, báo lỗi:

```
line 126: /home/user/openclaw/.env: Permission denied
```

Chạy ls -l /home/user/openclaw/.env thì thấy owner của file là root => cần chuyển owner về cho user bằng lệnh sudo chown -R user:user /home/user/openclaw

=> Đã cho phép chạy script, chạy được 100% script setup docker image.

Lỗi 2: Khi setup OpenClaw bằng onboarding wizard, gặp lỗi
```
Error: EACCES: permission denied, mkdir '/home/user'
```
Lỗi xảy ra có thể vì Dockerfile tạo một user node có id 1000 để chạy image nhưng id 1000 đã có user tên "user" sử dụng

**Qua kiểm tra thì thấy những lỗi này liên quan tới phân quyền người dùng của các thư mục trong Ubuntu Desktop và xung đột user => Chuyển sang sử dụng Ubuntu Server**

<h1 align="center">Setup OpenClaw trong môi trường Docker trên hệ điều hành Ubuntu Server</h1>

Thực hiện cài đặt Docker như trên.

Note: Sau khi đã cài đặt docker, để có thể chạy lệnh docker mà ko cần sudo thì cần tạo group là docker và thêm user hiện tại vào:
```
sudo groupadd docker
sudo usermod -aG docker $USER
```

Thực hiện cài đặt Openclaw như trên, chọn model, channel và test => thành công.

Note: do Ubuntu Server không có trình duyệt để mở link nên không thể authenticate trực tiếp để set model mà cần sử dụng API key.

**Skills của OpenClaw**

Khi cài đặt hay config OpenClaw, có thể thêm skills cho nó để mở rộng tính năng. Những skill đi kèm với onboarding wizard của OpenClaw để có thể tải trực tiếp từ đây:

- 1password: CLI của 1password để quản lý password và đăng nhập của cá nhân.

- apple-notes: quản lý Apply Notes qua CLI memo (MacOS)

- apple-reminders: quản lý Apple Reminders qua CLI remindctl (MacOS)

- bear-notes: tạo, search và quản lý Bear notes qua CLI

- blogwatcher: theo dõi cập nhật blog và luồng RSS/Atom

- blucli: tìm kiếm, chạy audio và nhạc.

- camsnap: capture frame hoặc clip từ camera RSTP/ONVIF

- clawhub: tìm kiếm và cài đặt skill được upload lên clawhub.com

- eightctl: kiểm soát thiết bị trợ ngủ Eight Sleep

- gemini: google gemini CLI để hỏi đáp nhanh, prompt ngắn

- github: tương tác với github

- gifgrep: tìm kiếm, query gif.

- gog: CLI google workspace để làm việc với các công cụ thuộc google như docs, sheet...

- goplaces: API của Google Places, tìm kiếm địa chỉ.

- himalaya: quản lý email qua phương thức IMAP/SMTP.

- imsg: quản lý tin nhắn trong iMessage

- mcporter: tương tác với server MCP

- model-usage: thông tin sử dụng model

- nano-banana-pro: gen ảnh bằng Gemini 3 Pro

- nano-pdf: edit PDF

- obsidian: tương tác và quản lý với Obsidian note.

- openai-whisper: speech-to-text.

- openhue: quản lý đèn Philips Hue

- ordercli:
