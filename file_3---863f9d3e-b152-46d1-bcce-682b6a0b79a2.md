<h1 align="center">Push lên một repo Github</h1>

Github có hỗ trợ tương tác với một repo thông qua Deploy Key. Deploy Key sử dụng key SSH để cấp quyền readonly hoặc write vào một repository nhất định.

- Giới hạn hoạt động trong một repo.

- Không cần cung cấp tài khoản cho OpenClaw.

- Có thể xóa key tại repo để hủy bỏ quyền tương tác.

Cách setup hệ thống với một repo:

B1: Tạo key SSH (VD: ssh-keygen -t ed25519 -C "openclaw-github" -f ~/.ssh/id_ed25519_openclaw). Không cần nhập passphrase.

B2: Bật ssh-agent và add key
```
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519_openclaw
```

B3: Copy key từ file key public (đuôi .pub) và nhập vào phần Deploy Key trong setting (Repo -> Settings -> Deploy keys -> Add deploy key). Bật allow write access nếu muốn OpenClaw có quyền viết (commit, push)

B4: Config host SSH để sử dụng key này:
```
Host github.com
HostName github.com
User git
IdentityFile ~/.ssh/id_ed25519_openclaw
IdentitiesOnly yes
```
Sau bước này, nên gia cố permission của các file ssh key:
```
chmod 700 ~/.ssh
chmod 600 ~/.ssh/config ~/.ssh/id_ed25519_openclaw
chmod 644 ~/.ssh/id_ed25519_openclaw.pub
```
Kiểm tra trạng thái kết nối github:
```
ssh -T git@github.com
```
B5: Trong repo local đã tạo, chạy:
```
git remote set-url origin git@github.com:<OWNER>/<REPO>.git
git push origin <branch>
```
Nếu chưa khởi tạo một repo local thì tạo folder mới bằng mkdir rồi git init.

<h1 align="center">Hosting file HTML trong GCloud</h1>

Dùng Caddy làm webserver, có thể sử dụng giải pháp khác như Nginx.

Chạy Caddy trong docker container.

B1: Cài đặt docker. Tạo một directory riêng để chứa các file.

B2: Tạo Caddyfile. VD;
```
loolanta.org {
    root * /srv
    file_server
}
```
Nếu không có domain, thay thế bằng IP public của VM (VD: http://34.9.78.196)

B3: Tạo một thư mục để chứa file html và đặt file cần host vào.

B4: Tạo file docker-compose.yml. VD:
```
services:
  web:
    image: caddy:latest
    container_name: caddy_demo
    ports:
      - "80:80"
      - "443:443"
    restart: unless-stopped
    volumes:
      - ./public:/srv
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config
volumes:
  caddy_data:
  caddy_config:
```
./public là thư mục chứa html

B5: Chạy docker compose và truy cập vào URL.
