<h1 align="center">Cập nhật OpenClaw</h1>

**Chạy lại script installer**
```
curl -fsSL https://openclaw.ai/install.sh | bash
```
- Nó sẽ tự phát hiện OpenClaw đã cài đặt trong hệ thống, cập nhật tại chỗ và chạy openclaw doctor để bảo trì.

- Thêm --no-onboard nếu không muốn chạy lại onboarding wizard (không có nhu cầu chỉnh sửa config)

- Script này thực hiện global install (cài đặt global, tải xuống và cài đặt qua package manager như npm hay pnpm)

Nếu lúc đầu thực hiện source install (cài đặt từ nguồn, cài đặt và build từ repo git), sử dụng:
```
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --install-method git --no-onboard
```

Note: cần để ý trước update:

- Cần nhớ lúc đầu đã cài đặt như thế nào, global (npm/pnpm) hay source (git clone)

- Gateway đang chạy như thế nào: foreground terminal hay supervised service (launchd/systemd)

- Backup config (~/.openclaw/openclaw.json), credentials (~/.openclaw/credentials/) và workspace (~/.openclaw/workspace)

**Update (Cài đặt global)**
```
npm i -g openclaw@latest
```
hoặc nếu sử dụng pnpm
```
pnpm add -g openclaw@latest
```
Sau đó chạy restart và bảo trì:
```
openclaw doctor
openclaw gateway restart
openclaw health
```

**Update (Cài đặt từ nguồn)**
```
openclaw update
```
hoặc có thể chạy thủ công như sau:
```
git pull
pnpm install
pnpm build
pnpm ui:build # auto-installs UI deps on first run
openclaw doctor
openclaw health
```

**Sau khi update thì luôn nên chạy openclaw doctor**

- Cập nhật config key và vị trí config file cũ.

- Kiểm tra sức khỏe gateway.

- Cập nhật service gateway cũ lên service Openclaw mới.

**Bật, tắt và restart gateway**
```
openclaw gateway status
openclaw gateway stop
openclaw gateway restart
openclaw gateway --port 18789
openclaw logs --follow
```

**Rollback về phiên bản cũ**
Global install:
```
npm i -g openclaw@<version>
```
hoặc
```
pnpm add -g openclaw@<version>
```

Source install: chọn 1 commit trong 1 ngày cụ thể
```
git fetch origin
git checkout "$(git rev-list -n 1 --before=\"2026-01-01\" origin/main)"
```
rồi tải lại dependency và khởi dộng lại
```
git fetch origin
git checkout "$(git rev-list -n 1 --before=\"2026-01-01\" origin/main)"
```
Nếu muốn quay về commit mới nhất:
```
git checkout main
git pull
```

Cả 2 phương pháp đều cần chạy openclaw doctor sau khi rollback:
```
openclaw doctor
openclaw gateway restart
```

<h1 align="center">Chuyển OpenClaw sang thiết bị khác</h1>

Dịch chuyển (Migrate) OpenClaw sang một hệ thống hay thiết bị khác mà không cần thực hiện setting onboard lại.

Mấu chốt: copy thư mục state (default: ~/.openclaw/, chứa config, auth, session và channel state) và workspace (~/.openclaw/workspace/, file liên quan tới agent như memory, prompt...)

**Trước khi bắt đầu**

1. Xác định thư mục state:

- Hầu hết sẽ nằm ở ~/.openclaw/

- Có thể là ~/.openclaw-\<profile>/

- Có thể kiểm tra bằng cách chạy openclaw status và tìm OPENCLAW_STATE_DIR

2. Xác định thư mục workspace:

- ~/.openclaw/workspace/ hoặc một thư mục custom do người dùng đặt lúc cài đặt.

- Là nơi chứa các file MEMORY.md, USER.md và memory/*.md.

3. Các thông tin sẽ được lưu trữ:

- Gateway config (openclaw.json)

- Auth profile, API key, OAuth token.

- Lịch sử session, trạng thái agent.

- Trạng thái channel (VD: Đăng nhập Whatsapp, session...)

- File workspace (memory, skills notes...)

Nếu chỉ copy workspace thì sẽ không lưu trữ session, credentials và channel login.

**Thực hiện migrate**

Bước 1: Back-up hệ thống cũ:

Dừng gateway:
```
openclaw gateway stop
```

Back-up thư mục state và workspace:
```
cd ~
tar -czf openclaw-state.tgz .openclaw
tar -czf openclaw-workspace.tgz .openclaw/workspace
```

Bước 2: Cài đặt OpenClaw trên hệ thống mới

- Thực hiện cài đặt như bình thường.

Bước 3: Copy file thư mục state và workspace vào hệ thống mới:

- Copy $OPENCLAW_STATE_DIR (~/.openclaw/) và workspace (~/.openclaw/workspace/)

- Các cách để copy:

    - sử dụng scp để chuyển file tar qua lại giữa các máy sử dụng ssh và extract.

    - rsync kèm ssh

    - external drive hoặc USB

- Sau khi copy, đảm bảo quyền sở hữu file đúng với user đang chạy gateway.

Bước 3: Restart gateway và kiểm tra lỗi:
```
openclaw doctor
openclaw gateway restart
openclaw status
```

<h1 align="center">Gỡ cài đặt OpenClaw</h1>

Uninstaller có sẵn:
```
openclaw uninstall
```
Hoặc xóa thủ công:
```
openclaw gateway stop
openclaw gateway uninstall
rm -rf "${OPENCLAW_STATE_DIR:-$HOME/.openclaw}"
rm -rf ~/.openclaw/workspace
npm rm -g openclaw
pnpm remove -g openclaw
bun remove -g openclaw
rm -rf /Applications/OpenClaw.app
```

Nếu gateway service vẫn còn nhưng không có CLI, thực hiện:
```
systemctl --user disable --now openclaw-gateway.service
rm -f ~/.config/systemd/user/openclaw-gateway.service
systemctl --user daemon-reload
```

<h1 align="center">Gateway</h1>

Gateway có thể hiểu là một phiên làm việc của OpenClaw.

- Một gateway có thể kết nối với nhiều channel cùng 1 lúc (Whatsapp, Discord...)

- Client để điều khiển OpenClaw (CLI, web UI) kết nối với gateway thông qua WebSocket (default: 127.0.0.1:18789)

- Mỗi host chỉ chạy 1 gateway.

Các thành phần trong kiến trúc Gateway:

Gateway (daemon)

- Là thành phần chính cho hoạt động của agent.

- Duy trì liên kết tới các API dịch vụ của bên thứ 3 như AI model, skills, các channel.

- Cung cấp API để người dùng điều khiển thông qua CLI hay Web UI hay tự động hóa.

- Thực hiện validation.

- Duy trì thông tin về các sự kiện như agent, chat, presence, health, heartbeat và cron.

Client (CLI, web UI)

- Là cách người dùng có thể điều khiển gateway.

- Mỗi client kết nối với 1 websocket connection.

- Gửi request tới gateway (health, status, send, agent, system-presence)

- Lắng nghe update về tick, agent, presence, shutdown.

Node (macOS, iOS, Android, headless server)

- Chỉ các thiết bị kết nối với gateway qua WebSocket server với role: node.

- Kết nối dựa trên danh tính của thiết bị, gateway lưu lại thông tin các node đã kết nối.

- Cung cấp các command như canvas.\*, camera.\*, screen.record, location.get.

WebChat

- UI tĩnh sử dụng API WebSocket của gateway để gửi và xem lịch sử chat.

- Trong các setup từ xa, kết nối qua SSH hoặc Tailscale giống các client khác.
